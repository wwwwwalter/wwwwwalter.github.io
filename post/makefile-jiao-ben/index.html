编译过程

1.预处理：头文件包含，宏展开，去注释
gcc -E main.c -o main.i
2.编译：做语法检查
gcc -S main.i -o main.s
3.汇编
gcc -c main.s -o main.o
4.链接
gcc main.o -o main
四步合为一步
gcc main.c -o main

第一层：显式规则
目标文件:依赖文件
[TAB]指令
第一个目标文件是最终目标
rm -fr main.i main.S main.o main
伪目标：.PHONY:

main:main.o
    g++ main.o -o main
main.o:main.S
    g++ -c main.S -o main.o
main.S:main.i
    g++ -S main.i -o main.S
main.i:main.cpp
    g++ -E main.cpp -o main.i

.PHONY:
clean_all:
    rm -fr main.i main.S main.o main
clean:
    rm -fr main

第二层：变量

  • =(赋值) 使用等号左边的变量时，取右边变量最终值
  • :=(赋值) 使用等号左边的变量是，取右边变量历史值
  • ?=(探测不到则赋值)
  • +=(追加)
    使用变量 $(变量名)
VIR_A = A
VIR_B = $(VIR_A) B
VIR_A = AA

all:
	echo ${VIR_B}
#AA B

VIR_A := A
VIR_B := $(VIR_A) B
VIR_A := AA

all:
	echo ${VIR_B}
#A B

TAR:=test
OBJ=circle.o cube.o test.o
CC:=gcc


$(TAR):$(OBJ)
	$(CC) $(OBJ) -o $(TAR)
test.o:test.c
	$(CC) -c test.c -o test.o
circle.o:circle.c
	$(CC) -c circle.c -o circle.o
cube.o:cube.c
	$(CC) -c cube.c -o cube.o

.PHONY:
clean:
	rm $(OBJ) $(TAR)

第三层：隐含规则
*.c *.o：所有的.c或者.o
%.c %.o：任意的.c或者.o

TAR:=test
OBJ=circle.o cube.o test.o
CC:=gcc



$(TAR):$(OBJ)
	$(CC) $(OBJ) -o $(TAR)
*.o:*.c
	$(CC) -c %.c -o %.o


.PHONY:
clean:
	rm $(OBJ) $(TAR)

第四层：通配符

  • $@所有的目标文件
  • $^所有的依赖文件
  • $<所有的依赖文件的第一个文件
  • $*不包含拓展名的目标文件名称
  • $+所有的依赖文件，以空格分开，并以出现的先后为序，可能包含重复的依赖
  • $?所有时间戳比目标文件晚的依赖文件，并以空格分开
TAR:=test
OBJ=circle.o cube.o test.o
CC:=gcc
RMRF:=rm -rf



$(TAR):$(OBJ)
	$(CC) $^ -o $@
%.o:%.c
	$(CC) -c $^ -o $@


.PHONY:
clean:
	$(RMRF) $(OBJ) $(TAR)
